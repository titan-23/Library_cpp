# 高速化に関する修正案
- メモリリアロケーションの排除。探索の最大ターン数や最大ビーム幅を用いて初期化時にメモリを確保し、動的なサイズ変更による処理の負荷を無くします。
- 時間ベースの動的ビーム幅調整。残りターン数と残り実行時間から毎ターンのビーム幅を算出し、制限時間超過を防ぎつつ探索手数を最大化します。
- アクション生成処理への閾値の伝搬。アクションの配列を返す処理をコールバック関数を用いる設計に変更し、アクションを一つ生成するたびにスコア計算とビーム追加を判定することで、無駄なインスタンス生成を削減します。
- 探索順序の最適化。探索木を更新する際、各親ノードが持つトップ候補のスコアで親ノードをソートし、有望な枝から優先して深さ優先探索を行うことで、閾値の更新を早め枝刈りを促進します。

# 枝刈りに関する修正案
- 相対スコアによる足切り。現在のビーム内のトップスコアから一定値以上離れている候補を破棄します。
- 軽量なヒューリスティックによる事前判定。状態の更新やハッシュ計算を行う前に、簡易的な計算で閾値を超えられないことが確定する操作を破棄します。
- 過去の履歴ハッシュによるループ検知。過去のターンで到達した状態のハッシュ値を保持し、同一状態への再到達を破棄します。

# 多様性確保に関する修正案
- 親ノードからの派生数制限。一つの親ノードから次ターンのビームに残せる子ノードの数に上限を設けます。
- 状態の抽象化による重複排除。盤面の微小な違いを無視した大まかなハッシュ値を計算し、本質的に似た状態を同一視して重複を排除します。
- グループ化ビームサーチ。候補を特定の条件でグループに分け、各グループから一定数ずつ候補を残します。
- 兄弟ノードへのソフトペナルティ。同一の親から派生した二番目以降の候補に対して、スコアにペナルティを加算してからビームに追加します。
- 生存期間に基づく救済。親から分岐してからのターン数が短い新しく生成された経路を、優先的に数枠確保します。

# デバッグおよび可視化に関する修正案
- 探索木の出力機能。各ターンのノードのスコア、ハッシュ値、親ノードの識別子などをファイルに出力し、外部のツールで探索の過程を分析可能にします。

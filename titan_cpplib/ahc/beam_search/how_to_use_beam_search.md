### 1. 実装上の制約と高速化の指針
以下の指針に従って実装してください。

- Action構造体の設計とメモリ管理
  - スケルトンコード内のメンバ変数は一例です。対象問題の操作を表現するために必要な変数を自由に変更および追加してください。
  - Action構造体は大量に生成およびコピーされます。ロールバックに必要な情報は上書きされて失われる前の情報のみとし、変数の型は必要十分な最小サイズを選択してサイズを最小化してください。
- スコア計算の打ち切り(最小化問題)
  - 提供されたビームサーチはスコアの最小化を行います。`try_op`関数に渡される`threshold`は現在のビーム内で最も大きいスコアです。
  - 差分計算の過程で次のスコアが`threshold`以上になることが確定した時点で計算を打ち切り、タプルで `{INF, 0, false}` を返してください。この場合その`Action`が`apply_op`されることはないため、Actionへのロールバック情報の保存処理もスキップしてください。
- Zobrist Hashの適切な設計
  - Zobrist Hashによる差分更新を実装してください。ハッシュ化する対象は盤面の構造などその後の状態遷移やスコア計算に影響を与える要素のみに限定してください。
  - 乱数テーブルを事前に初期化し、XOR演算を用いて高速にハッシュ値を更新してください。
- 状態の差分更新
  - State内の盤面配列などをディープコピーすることは禁止します。必ず変更があった箇所のみを計算・更新してください。
- 不要な`Action`の枝刈り
  - `get_actions`関数では、明らかにスコアが悪化する無駄な手や、直前の操作を打ち消すだけの手を生成しないでください。

### 2. 各関数の仕様と実装要件
別途提供するスケルトンコード内にある各関数について、以下の要件を満たすように実装してください。関数のシグネチャは絶対に変更しないでください。

#### Action構造体
- 目的: 状態遷移を表す操作パラメータと、ロールバックに必要な情報を保持します。
- 要件: 状態を元に戻すための情報をメンバ変数として追加してください。スコアやハッシュの計算に時間がかかるのであれば保持しても構いませんが、apply,rollback内でそれほど時間がかからずに差分更新できるのであればそちらで再計算することを強く推奨します。

#### State::init関数
- シグネチャ: `void init()`
- 目的: 探索開始時の初期状態を構築します。
- 要件: 盤面、スコア、ハッシュ値の初期化を行ってください。Zobrist Hash用の乱数テーブルの初期化もここで行うか、事前に計算済みのものを読み込んでください。

#### State::try_op関数
- シグネチャ: `tuple<ScoreType, HashType, bool> try_op(Action &action, const ScoreType threshold) const`
- 目的: 現在の状態に操作を適用した場合の、次状態のスコア、ハッシュ値、および状態が終了したかを計算します。
- 要件: const関数であるため変数は変更しないでください。スコアとハッシュ値は差分計算で求めてください。計算が最後まで完了した場合のみ、計算結果とロールバックに必要な情報を引数の`action`に記録してください。3つ目の戻り値には、その操作によって探索が終了状態に達したかどうかを表す真偽値を返してください。

#### State::apply_op関数
- シグネチャ: `void apply_op(const Action &action)`
- 目的: 状態を次の状態へ進めます。
- 要件: actionに保存された情報を用いて、`State`の変数をインプレースで更新してください。この関数は`try_op`を通過した合法なActionに対してのみ呼ばれるため、再度のバリデーションや再計算は不要です。

#### State::rollback関数
- シグネチャ: `void rollback(const Action &action)`
- 目的: 状態を一つ前の状態に戻します。
- 要件: `action`に保存された情報を用いて、`apply_op`実行前の状態に完全に復元してください。ここでも差分のみを巻き戻してください。

#### State::get_actions関数
- シグネチャ: `void get_actions(vector<Action> &actions, const int turn, const Action &last_action, const ScoreType threshold) const`
- 目的: 現在の状態で実行可能な合法手を列挙します。
- 要件: 可能な操作を生成し、actionsに追加してください。actionsのメモリ再確保を防ぐため、事前に最大生成数がわかる場合はreserveを利用してください。※ターン0の呼び出し時など、last_actionにダミー値が含まれる場合があるため、無条件でlast_actionのメンバを参照して不正アクセスを起こさないよう注意してください。

#### State::print関数
- シグネチャ: `void print() const`
- 目的: デバッグ用に表示する
- 要件: 現在の状態を標準エラー出力に表示する。計算量は多少大きくなっても良いので、ユーザーが見やすいような表示をしてください。また、最適化本体に関係はないので、print関数のために余計な情報は持たないようにしたいです。

#### State::get_state_info関数
- シグネチャ: `string get_state_info() const`
- 目的: 互換用関数
- 要件: return "{}"; としてください。

#### search関数
- 目的: ビームサーチ本体を実行する
- 要件
  ```cpp
  vector<Action> search(flying_squirrel::BeamParam &param, const bool verbose=false, const string& history_file = "") {
      flying_squirrel::BeamSearchWithTree<ScoreType, HashType, Action, State, INF> bs;
      return bs.search(param, verbose, history_file);
  }
  ```
  上のコードをそのまま書いてください。

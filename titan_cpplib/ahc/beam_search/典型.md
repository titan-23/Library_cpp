# AHC典型

## 大方針

- 問題文を言い換える
- 問題を分割する

- 焼きなまし or ビームサーチ
  - 文脈が強いか否か
    - 強くても焼ける場合がある
      - ローリングホライズン
  - 上手く下界を見積もれるとビムサの方が強いことも

- 問題を簡単な構造にする
  - グラフ構造への帰着
    - 木構造とみなす
      - 木構造は焼きやすめかも？
      - 最短経路、最小全域木、シュタイナー木
    - DAGとみなす
      - DAGも焼きやすそう
    - 最小費用流、二部マッチング等のグラフアルゴリズム
  - グリッドに限定する
  - 初めは粗く、段階的に細かくする
  - 連結成分数、転倒数、LIS、マンハッタン距離、バウンディングボックス、重心、次数、、、

- 最終形の特徴を考える
  - 長時間焼く or レプリカで並列化して良い解を見つける
  - ゴール状態からの逆算
    - 最終的な目標状態から1回の操作、あるいは2回の操作を巻き戻した状態を構築し、その状態が持つ性質を観察する
      - (AHC064) 完成直前の盤面は、複数の正しい連続部分が異なる場所に存在し、それらを連結するだけの状態となるので、連続する部分構造の構築が目標達成の中間目標になることが分かる。
  - 手順を増加させる要因の分析
      - (AHC064) 要素が目標に近い位置にあっても、その前後に不適切な要素が存在すると退避のための余分な操作が発生する。この事実から、要素の絶対的な座標よりも隣接する要素との関係性の方が手数の削減において重要であることが分かる。
  - 手動シミュレーションの言語化
    - 規模の小さい問題を人間が手作業で解き、その際の意思決定の基準を言語化する
      - (AHC064) 人間が操作を行う際、個々の要素を独立して動かすのではなく、要素の塊を構築しそれを単位として動かす戦略を自然に採用します。この無意識に認識している塊という概念を数値化することで、連結成分という特徴を定義できます。
  - 保存すべき構造を考える
      - (AHC064) 探索の過程において、一度構築した後に破壊するべきではない構造を定義します。正しい順序で連結された要素群は、それ以降の操作で分断する論理的な理由がありません。維持するべき構造を定義して評価関数に組み込むことで、構築済みの構造を保護する状態が高い評価を得るようになります。

- 制約緩和
  - 評価関数が難しいのでなるべくやりたくない
  - 特に、valid でない状態から valid に戻りにくいとき、緩和してペナルティをかけない方が良い気がする

### ビームサーチ
- 適切な世代を考える
  - (AHC063) 次の一手をとる or 1マス移動
  - 盤面に進行度を保持させる
    - (AHC064) 交差制約のため、一方向の段階的な状態遷移をさせる
  - チューニングすれば基本ミクロな一手の方が強いはず
    - 評価関数が難しいイメージ
- 重複除去
  - 盤面をハッシュ化して探索済み盤面を除去する
  - 厳密には同一盤面でなくても、同一視したいものを除去できれば良い
    - (AHC063) 餌の位置と蛇の頭の位置で十分。蛇の胴体をハッシュ化すると多様性が損なわれる。
- 高速化
  - 自明な下界で閾値との枝刈り(強い)
  - 入力がランダムであることを活かし、一定以下は枝刈ってしまう
- スコアに乱数を入れて多様性確保

### 焼きなまし
- 初期解
  - 重要なときとそうでないときがある
- 近傍定義
  - 具体的な近傍
    - 列
      - 1点挿入, 1点更新
      - 2点swap, range swap, suffix swap(AHC049)

### 評価関数
- 最終的な目標状態との差分を数値化して定義する
- スコアを悪化させる要因はなにか考える

## プロンプト
```
以下の競技プログラミングの問題文を読み、ストーリー設定を一切排除して、純粋なデータ構造（スタック、キュー、グラフ、配列など）と、許される操作、満たすべき制約の言葉だけで再定義してください。
```

```
一気に目標状態を作るのは困難です。初期状態と目標状態の間に設定すべき『中間目標となる盤面（こういう形が作れたら嬉しいという状態）』のアイデアを、理由とともに挙げてください。
```

```
この問題において、十分賢い人間であればどのようなヒューリスティクスで最適解を見つけると思いますか？
人間が直感的に行っているその判断基準を、アルゴリズムとして実装できるルールや評価値に言語化してください。
```

```
この問題に対する「大方針（全体のアプローチ）」の候補を、全く異なるアプローチで3〜4つ提案してください。
(焼きなまし、状態を工夫したビームサーチ、貪欲法等々)

各アプローチについて、以下のフォーマットで解説してください。
【アプローチ名】
概要：
この問題の性質に合っている理由：
懸念される最大のボトルネック・弱点：

提案されたアプローチの比較を行いたいです。
理論上のスコア上限が最も高そうなのはどのアプローチですか？
推奨アプローチを1つ選び、その理由を教えてください。
```

```
この問題の評価関数を考えます。以下の3つの観点から特徴量を提案してください。
- 局所的な盤面の良さ
- 次のターンの操作のしやすさ
- 目的達成の阻害定量化
```

```
この問題で許可されている操作について、以下の2点を分析してください。
1. 可逆性：ある操作を行った後、それを完全に無かったことにする（元の状態に戻す）操作手順は存在しますか？存在する場合、何手かかりますか？
2. 不変量：操作を何度繰り返しても「絶対に変化しない値」や「保存されるパリティ（偶奇性など）」、「要素間の相対的な関係」はありますか？
これらを踏まえ、無駄な探索を省くための「枝刈りの条件」を提案してください。
```

```
この問題において、手数を増加させている（またはスコアを悪化させている）最も厄介な制約やペナルティを1〜2個特定し、それらを「完全に無視してよい（コスト0、容量無限大、ワープ可能など）」とした『緩和問題』を考えてください。
その緩和問題において、現在の状態から目標状態（または最適解）へ到達するための最も単純なロジックと、その際にかかる最小手数（または最大スコア）の計算式を答えてください。
さらに、その計算結果を元の問題における「理想状態までの距離（下界）」として、評価関数に組み込むアイデアを提案してください。
```

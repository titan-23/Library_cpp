あなたは競技プログラミングのヒューリスティックコンテストに参加する優秀なプログラマです。
別途提供する焼きなまし法のライブラリ実装と対象問題の仕様に基づき、Stateクラスを実装してください。

### 1. 実装上の制約と高速化の指針
パフォーマンスの最適化とフレームワークの仕様順守が最重要です。以下の指針に従って実装してください。

- フレームワークの制約順守
  - テンプレートにある `#include` や `namespace sa` の枠組みは改変しないでください。
  - クラス名、構造体名、変数名、各メソッドのシグネチャは変更しないでください。
  - State および Changed のデフォルトコンストラクタは削除しないでください。
  - グローバル変数や関数内の static 変数の新規定義は禁止します。必要な状態はすべて State または Changed のメンバとして持たせてください。
- 状態の管理とコピーの回避
  - 探索ループ内において State クラス自体のコピーは発生しません。単一のインスタンスを差分更新で使い回します。盤面全体を再計算したり配列全体をコピーする処理は避け、変更が影響する局所的な範囲のみを再計算し、スコアの差分を加減算してください。
- 作業領域の活用
  - changed は動的メモリ確保を避けるための作業領域として活用してください。前回のデータが残っているため、使用するメンバ変数は必ず上書き更新してください。
- スコア計算の打ち切り
  - 差分計算の過程でスコアの悪化量が threshold を明らかに超えることが判明した場合は計算を打ち切り、is_valid = false にして直ちにリターンしてください。盤面を書き換える前に遅延評価で判定する実装を推奨します。
- スコアの最小化と二重反転の禁止
  - 実行側はスコアが小さいほど良いとして動作します。最大化問題の場合は計算したスコアにマイナスをつけて保持してください。
  - 最大化問題の場合でも get_score() は return score; のままとし、内部で再度マイナスをつける二重反転は行わないでください。get_true_score() には本来の生のスコアを返す処理を記述してください。

### 2. 各構造体および関数の仕様と実装要件

#### State::Param構造体
- 目的: 焼きなましの温度パラメータを保持します。
- 要件: 対象問題のスコアスケールに合わせて start_temp および end_temp の初期値を設定してください。

#### State::Changed構造体
- 目的: 1手分の遷移の差分情報と変更前の状態を記録する作業領域です。
- 要件: TYPE_CNT に実装する近傍操作の種類の総数を設定してください。実行した操作の種類を記録する type 変数に加え、ロールバックに必要な情報を保持する変数を追加してください。

#### State::Result構造体
- 目的: 探索中に発見した最良解の状態を保持し、最終的な解答を出力します。
- 要件: Result は最良解が更新されるたびに生成およびコピーされます。処理負荷を避けるため、盤面全体のディープコピーは行わないでください。print メソッドでの解答出力に必要となる最小限の情報のみをメンバ変数として持たせてください。

#### State::init関数
- シグネチャ: void init(uint32_t seed)
- 目的: 探索開始時の初期状態を構築します。
- 要件: 状態配列の初期化を行い、初期解のスコアを計算して score に代入してください。

#### State::modify関数
- シグネチャ: void modify(const int64_t iter, const ScoreType threshold, const double progress)
- 目的: 近傍操作を選択し、状態とスコアを更新します。
- 要件: sarnd を用いて近傍操作を選択してください。操作の種類を0始まりのインデックスで changed.type に代入し、ロールバックに必要な情報を changed に保存してください。ルール違反の場合は is_valid = false としてリターンしてください。

#### State::rollback関数
- シグネチャ: void rollback()
- 目的: 遷移が棄却された場合に状態を元に戻します。
- 要件: changed に保存された情報を用いて、状態と score を modify 実行前の状態に復元してください。

#### State::advance関数
- シグネチャ: void advance()
- 目的: 遷移が採択された場合に状態を確定させます。
- 要件: modify で遅延評価を採用した場合は、ここで状態の書き換えを確定させてください。すでに更新済みの場合は空で構いません。

#### State::print関数
- シグネチャ: void print() const
- 要件: 盤面の状態を出力します。

#### State::get_result関数
- シグネチャ: Result get_result() const
- 目的: 現在の状態を Result 構造体として返します。
- 要件: 現在の状態から解答の出力に必要な最小限の情報を抽出し、Result オブジェクトを生成して返してください。

### 3. 外部ライブラリ
#### 乱数
- `sarnd` (`thread_local titan23::Random`): 固有の高速なXorshift乱数生成器です。以下の仕様に従って使用してください。
  - `sarnd.random()`: [0.0, 1.0] の実数を返す。
  - `sarnd.randint(end)`: [0, end] の整数を返す（両端含む）。
  - `sarnd.randint(begin, end)`: [begin, end] の整数を返す（両端含む）。
  - `sarnd.randrange(end)`: [0, end) の整数を返す（endを含まない）。
  - `sarnd.randrange(begin, end)`: [begin, end) の整数を返す（endを含まない）。
  - `sarnd.shuffle(vec)`: vectorをインプレースにシャッフルする。

### 4. 効率よく高速なコードを書くためのヒント
- 差分更新: `modify` 内で盤面全体を再計算したり、配列全体をコピーしたりしないでください。変更が影響する局所的な範囲のみを再計算し、スコアの差分を加減算してください。

以上の仕様、ルール、およびヒントに従い、出力してください。
